# Feature Delivery Playbook

## Steps

1. Define user problem.
2. Identify owning code path.
3. Document data model impact.
4. Identify trust boundaries.
5. Make smallest implementation.
6. Validate behavior.
7. Update docs.
8. Report remaining risk.

## Required Artifacts

- Feature summary
- Data impact note
- Test/validation result
- Docs update when user behavior changes
