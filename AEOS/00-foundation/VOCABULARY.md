# Vocabulary

## Canonical Terms

- **Invariant**: A condition that must remain true for the system to be correct.
- **Boundary**: A place where trust, state, or ownership changes.
- **Evidence**: Data that supports a claim but may still require interpretation.
- **Confidence**: A score that represents extraction reliability, not truth.
- **Mutation**: A write that changes durable state.
- **Idempotency**: The property that repeated execution does not create duplicate effects.
- **Rollback**: A concrete path to undo or safely recover from a decision.
- **Trust boundary**: A boundary where input must be validated before use.
- **Source of truth**: The durable system that owns a fact.

## Reserved Terms

- **Secure**: Use only when a concrete security property is enforced.
- **Guaranteed**: Use only when tests, constraints, or mechanisms enforce the claim.
- **Real-time**: Use only when latency expectations are defined.
