## Definition Of Ready

- [ ] Controlling code/system path is known.
- [ ] Data ownership, interfaces, and trust boundaries are known.
- [ ] Requirements and non-requirements are explicit.
- [ ] Principal failure modes and irreversible risks are explicit.
- [ ] Acceptance criteria and cheapest discriminating check are explicit.
- [ ] The decision state permits the proposed next action.

## Definition Of Done

- [ ] Requirements are mapped to evidence.
- [ ] Smallest safe change or experiment completed.
- [ ] Focused and repository-required checks pass.
- [ ] Security, privacy, accessibility, operations, and data risks reviewed as applicable.
- [ ] Documentation, decision log, and project memory match reality.
- [ ] Rollback limits and residual risks are documented.
