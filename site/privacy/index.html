Privacy policy

Community Ledger is a local-first Android beta. It records event money information but does not collect, hold, invest, transfer, or process money.

Publisher: Community Ledger is independently published by Gopi Banoth. Private support and privacy email: banothgopikrishna19@gmail.com. Public identity: GitHub and LinkedIn.

Data stored on your device

The app can store event details, member contact details entered by an organizer, transaction records, payment references, UPI IDs, reviewed receipt JSON, a self-declared local email label, preferences, and non-sensitive interruption markers.

The Room database, receipt JSON, and preferences use app-private storage and are excluded from Android backup and device-transfer rules.

Receipt images and OCR

Selected or shared receipt images are read on the device with Google ML Kit Latin and Devanagari text recognition. OCR can be wrong. Extracted values do not affect totals until a person reviews and saves them.

The app does not send receipt images to a Community Ledger server. It stores reviewed JSON evidence rather than a cloud OCR result.

Network use

The beta has no account, synchronization, advertising, crash-reporting, or transaction server.

Google's ML Kit Android disclosure says its SDK collects limited device and app information, bundled per-installation identifiers, performance, API configuration, input/output size, feature-event, and error metadata for diagnostics and usage analytics. Google says this metadata is HTTPS-encrypted and is not transferred to third parties. Receipt images and recognized text are processed on-device.

The update checker contacts a static HTTPS release manifest only when a user taps Check for updates. GitHub Pages and its infrastructure can receive ordinary connection metadata such as IP address, request time, and user agent. There is no automatic background update check.

Read Google's ML Kit Android data disclosure.

Sharing and event-copy links

Android share actions send only content selected by the user. An event-copy link can contain an opaque copy key, event title, expiry, visibility marker, and organizer's self-declared local email label. Recipients can forward it.

A link does not authenticate the organizer, enforce private access, or connect devices to one synchronized ledger. Each phone receives an independent local event shell.

No sale or backend access

Community Ledger does not sell personal information. Because there is no Community Ledger backend, the project cannot view or retrieve ledger content stored on a user's device.

Retention and deletion

Local data remains until the user deletes an event, clears app data, or uninstalls the app. Uninstalling deletes the ledger from that device. There is currently no export, restore, or account recovery.

Permissions

The app does not request camera, location, contacts, microphone, SMS, call-log, or broad photo-library permission. Android grants temporary access only to selected or shared receipt content. Internet access is used for a manual update request and opening hosted information.

Organizer responsibilities

Organizers should collect only information needed for their event, obtain any required consent, avoid unnecessary contact or payment details, and follow laws applying to their event and fundraising activity.

Security limits

App-private storage, reviewed OCR, duplicate checks, and validated writes reduce specific risks. They do not make a device tamper-proof. Rooted devices, debug builds, forwarded links, shared screenshots, or physical access can expose data.

Your choices

  • Do not select or share receipt images.
  • Avoid entering optional contact details.
  • Delete events and associated local records.
  • Clear app data or uninstall the app.
  • Stop using event-copy links.

Contact and changes

Email banothgopikrishna19@gmail.com for private support or privacy questions. Use the public project support page only for non-sensitive bugs. Do not post receipts, personal or financial data, passwords, signing keys, or verification codes there. Material policy changes will update the effective date and release notes.
